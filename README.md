# **全量批量修改照片/视频 EXIF 与重命名（备份版）**

## **功能概览**

- 批量处理图片和视频文件，支持：
  - 图片：JPG/JPEG/PNG/HEIC/WebP
  - 视频：MP4/MOV
- 根目录仅备份一次，可选
- 支持目录名解析：
  - 格式：`YYYYMMDD-注释`
  - 图片/视频文件会使用 **原始 EXIF 或 MediaCreateDate** 作为优先时间
  - 如果没有原始时间 → 使用目录日期 + 12:00 自动生成
- 文件重命名：
  - 格式：`IMG_YYYYMMDD_HHMMSS.ext`
  - 重复文件名会自动加序号处理
- 多线程处理，提升处理速度
- 进度条实时显示处理进度
- 处理完成后日志输出 `[全部处理完成]`，不弹窗
- 只使用系统 PATH 下的 exiftool，不打包内置 exe
- GUI 界面，操作简单，通过点击选择目录，不支持拖拽
- 程序会改变图片的创建时间和最后修改时间

## **使用方法**

### **方式 A：直接使用 exe（推荐）**

1. 确保已安装 [exiftool](https://exiftool.org/)，并在系统 PATH 中设置
2. 下载或复制 `full_exif_gui_backup_sync.exe` 到任意文件夹
3. 双击 exe 打开 GUI
4. 点击 **浏览** 按钮选择要处理的根目录，或者复制路径
5. 勾选“是否备份原文件”（可选）
6. 点击 **开始处理**，GUI 显示进度条
7. 文件处理完成后，日志末尾显示 `[全部处理完成]`

✅ 不需要安装 Python 或其他依赖，适合普通用户直接使用

------

### **方式 B：使用 Python 脚本运行或打包 exe**

#### **1. 安装依赖**

```bash
pip install PySimpleGUI
```

- 确保系统 PATH 中有 exiftool
- Python 3.7 及以上版本

#### **2.A 运行脚本**

```bash
python full_exif_gui_backup_sync.py
```

- 脚本界面与 exe 一致

#### **2.B 打包成单文件 exe（推荐）**

```shell
pyinstaller -F -w full_exif_gui_backup_sync.py
```

- `-F`：生成单文件 exe
- `-w`：GUI 模式，不显示控制台窗口
- 生成的 exe 会在 `dist/` 目录下
- 运行 exe 时仍需确保系统 PATH 有 exiftool

------

## **文件命名与时间规则**

- **文件名生成规则**：`IMG_YYYYMMDD_HHMMSS.ext`
- **时间优先顺序**：
  1. 图片：原始 EXIF `DateTimeOriginal`
  2. 视频：`MediaCreateDate`（若有）
  3. 无原始时间：使用目录日期 + 12:00，秒数根据文件顺序自动递增
- **重复文件名处理**：
  - 自动在文件名后加 `_1`, `_2` 等序号，保证文件名唯一

D:\Temp

```shell

│  └─ ...
├─ 20130217-春节
│  ├─ IMG_20130217_120000.jpg
│  ├─ IMG_20130217_120001.png
│  ├─ IMG_20130217_120002.mp4
│  └─ ...
```

## **备份说明**

- 备份仅在根目录创建一次
- 文件会完整复制到 `BackupYYYYMMDD_HHMMSS_微秒` 目录
- 脚本会自动检查磁盘剩余空间，如果空间不足会提示错误并停止处理

------

## **多线程处理**

- 默认最多同时处理 8 个文件（可修改 `MAX_WORKERS` 参数）
- 每个文件在单独线程中调用 exiftool
- 进度条通过线程安全队列实时更新

------

## **注意事项**

- **拖拽目录不可用**，请使用 **浏览按钮选择目录**
- 生成的 EXE 大小约 24MB，包含 Python 解释器和依赖库，是正常现象
- 系统必须安装 exiftool 并加入 PATH
- 视频文件只读取 `MediaCreateDate` 和 `TrackCreateDate`，若视频没有这些时间，则使用目录日期
- 处理完成不会弹窗，只会在日志输出 `[全部处理完成]`

------

## **系统兼容**

- Windows 7 及以上版本
- Python 3.7+ 环境（打包成 exe 可直接运行，无需安装 Python）
- 图片和视频的 EXIF/MediaCreateDate 修改依赖 exiftool 正常运行

## **总结**

- 脚本方便用户批量修改照片和视频的时间和文件名
- 支持多线程、备份、原始 EXIF 时间优先、重复文件名自动处理
- GUI 简单直观，进度条实时显示
- 不依赖拖拽操作，只需点击选择目录即可
- 完全依赖系统 PATH 下的 exiftool
